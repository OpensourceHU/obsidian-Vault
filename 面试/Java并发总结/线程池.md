1. 池化复用是为了避免任务提交后线程的频繁创建和任务完成后的频繁销毁, 也是一个用空间换时间的做法
2. 同时, 线程池相当于一个执行者, 程序员将业务代码逻辑 和 具体线程调度分开, 也是一种解耦

线程池一共就几个问题:
1. 构建线程池的参数, 如何调整核心线程数和最大线程数

2. 线程池有新任务提交时, 执行顺序


### 线程池的参数
